# Copilot Coding Rules for AI Growth Planner

- Always use clean architecture.
- Use modular NestJS structure for backend.
- Web uses Next.js App Router.
- Mobile uses React Native with functional components.
- Always generate TypeScript types.
- Prefer composition over inheritance.
- Always separate UI from logic (hooks in /hooks).
- Follow file naming conventions: kebab-case.
- Use async/await.
- For backend modules, always generate:
  - controller
  - service
  - module
  - dto
  - types
